# ArcanisExperiments

A safe environment for experimental ideas at ARCANIS LAB.

## Purpose
A sandbox for trying out speculative, exploratory, and proof-of-concept work
**without affecting any stable project**. Failed ideas are welcome here.

## Categories
| Folder | What lives here |
|--------|-----------------|
| `ai/` | AI / ML experiments, models, pipelines |
| `hardware/` | Microcontrollers, circuits, sensors, firmware |
| `kernel/` | OS / kernel subsystems, drivers, scheduling |
| `language/` | Programming language & compiler experiments |
| `ui/` | UI/UX prototypes and interaction studies |
| `templates/` | Reusable experiment scaffolds |
| `research-notes/` | Free-form research notes & reading logs |
| `docs/` | Additional documentation |

## How to start an experiment
1. Pick a category folder.
2. Copy the matching template from `templates/` into that folder.
3. Rename it `<experiment-name>-<YYYY-MM-DD>.md`.
4. Fill in the template and create any code under the same folder.
5. Keep a running log; archive when done.

## Rules
See [RULES.md](RULES.md). In short: experimental code only, never touch
stable projects, and document everything.
