# Experiment Workflow

A consistent lifecycle for every experiment.

## 1. Idea
- Copy the relevant template from `../templates/`.
- Set `status: idea` and fill the `Hypothesis` / `Goal`.

## 2. Active
- Set `status: active`.
- Build in the matching category folder.
- Update the `Log` with dates and decisions.

## 3. Paused
- Set `status: paused` if blocked or deprioritized.
- Leave enough context to resume later.

## 4. Archived
- Set `status: archived`.
- Summarize conclusion; keep artifacts for reference.

## Tracking
Maintain a manual index in the category folder or `docs/` listing each
experiment, its status, and a one-line summary.
