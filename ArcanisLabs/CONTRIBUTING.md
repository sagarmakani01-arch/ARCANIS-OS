# Contributing to ArcanisLabs

Thank you for your interest in contributing. ArcanisLabs is a long-term research and development ecosystem, and every contribution brings us closer to ArcanisOS.

## How to Contribute

### 1. Understand the Vision
Read the [Vision document](./00_Documentation/Vision/Vision.md) and [Architecture](./00_Documentation/Architecture/Architecture.md) to understand where we're going and how the pieces fit together.

### 2. Pick a Project
Each project has its own README with a status indicator:
- **Foundation** — Ready for implementation
- **Planning/Design** — Needs research before coding
- **Research** — Requires deeper study
- **Active** — Currently being built

Start with Foundation-phase projects (24-30, 00) before moving to higher layers.

### 3. Follow the Standards
Read the [Development Standards](./00_Documentation/Standards/DevelopmentStandards.md) for:
- Code style and naming conventions
- Testing requirements
- Commit message format
- Branch strategy

### 4. Development Workflow
1. Fork or branch from `main`
2. Read the project's README and docs
3. Set up your development environment
4. Run existing tests to verify setup
5. Write tests first (TDD recommended)
6. Implement your changes
7. Run all tests
8. Submit a pull request

### 5. Pull Request Guidelines
- Keep PRs focused on a single change
- Link to any related issues
- Include before/after if changing behavior
- Update documentation for API changes
- Ensure all tests pass

## Communication
- Use issues for bug reports and feature requests
- Use pull requests for code contributions
- Be respectful and constructive

## What We Need Help With
- **Documentation** — Improving clarity and completeness
- **Testing** — Writing tests and test infrastructure
- **Research** — Exploring technologies and approaches
- **Implementation** — Building the components

All contributors are expected to follow our [Code of Conduct](./CODE_OF_CONDUCT.md).
