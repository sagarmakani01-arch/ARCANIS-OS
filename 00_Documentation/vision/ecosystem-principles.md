# Ecosystem Principles

**Path:** `vision/ecosystem-principles.md`  
**Version:** 0.1.0  
**Status:** Draft

---

## 1. Modular Sovereignty

Each project in the Arcanis ecosystem is independently versioned, documented, and governed. No single project holds a monopoly on the ecosystem's direction. Projects communicate via well-defined contracts, not shared implementations.

## 2. Contract-First Integration

Cross-project interaction is defined by formal interface contracts (schemas, protocols, APIs). Contracts are versioned and evolve through a deprecation process — never breaking changes without notice.

## 3. Progressive Disclosure

Complexity is layered. Beginners see simplicity; experts can drill into every detail. Documentation, APIs, and system internals follow this principle at every level.

## 4. Fail Closed, Fail Audible

Security defaults to the most restrictive mode. When failures occur, the system produces detailed, actionable diagnostics. Silence is not an acceptable failure mode.

## 5. Data Gravity with Portability

Data stays where it is created unless explicitly moved. But every data format has a documented, versioned schema, and every storage layer supports export to open formats.

## 6. Dogfood Everything

Every Arcanis project must be self-hosting or self-consuming. The OS runs on itself. The compiler compiles itself. The documentation documents its own process.

## 7. Research-Driven Engineering

All engineering decisions must be informed by research. Prototypes precede products. Benchmarks precede optimizations. Understanding precedes implementation.

## 8. Inclusive by Design

The ecosystem supports multiple languages, accessibility standards, and diverse hardware. "Works on my machine" is unacceptable — the ecosystem must work everywhere.

---

*Principles are not rules. They are the values we weigh when rules conflict.*
